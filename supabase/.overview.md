# supabase Overview

`supabase/migrations` contem a evolucao do banco do app de prospeccao por e-mail. O modelo atual usa `workspace_id`, RLS e tabelas compartilhadas por tenant.

Cuidados:

- Nao edite migrations ja aplicadas salvo pedido explicito.
- Para mudancas de schema, adicione uma nova migration com timestamp.
- Depois de mudar o banco, atualize `types/database.ts` quando houver fluxo de geracao disponivel.
- Consulte `product-brief.md` para decisoes de produto, compliance, Google OAuth, campanhas e engine de e-mail.

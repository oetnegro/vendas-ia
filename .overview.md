# Repo Overview

Este repo combina duas superficies que precisam continuar separadas em intencao:

- **Landing publica:** raiz `/`, principalmente `app/page.tsx`, SEO global em `app/layout.tsx`, contexto em `landing-page-brief.md`.
- **App autenticado (Vendas+IA Email Prospecting):** rotas em `app/(app)`, auth em `app/(auth)`, APIs em `app/api`, contexto em `product-brief.md`.
- **Banco e migrations:** `supabase/migrations` (8 migrations — ver `supabase/migrations/.overview.md`), tipos em `types/database.ts`.
- **Scheduler MVP:** workflow n8n `Vendas+IA Email Engine Cron` chama `/api/cron/email-engine` a cada 5 min enquanto o projeto está no Vercel Hobby. Migrar para Vercel Cron subdiario quando subir para Vercel Pro.

## Stack

- Next.js 16 (App Router) + React 19 + Tailwind 4
- Supabase (Auth + Postgres com RLS multi-tenant via `workspace_id`)
- Gemini (`gemini-2.5-flash`) via `GEMINI_API_KEY` server-side — IA do produto, chave única para todos workspaces
- Gmail API (envio, sync de threads) via Google OAuth do usuário
- Google Calendar API (criação de eventos quando IA agenda reunião)
- Resend (formulário de contato da landing — `RESEND_API_KEY`)
- Deploy: Vercel (projeto `vendas-ia-landing`, plano Hobby hoje)

## Decisões fechadas críticas (não óbvias no código)

- **Multi-tenant via `workspace_id` + RLS.** Nunca tabela por cliente. `workspace_members` + `is_workspace_member()` governam acesso.
- **`SUPABASE_SERVICE_ROLE_KEY` e `GEMINI_API_KEY` jamais no browser.** Só server-side / `.env.local`.
- **`NEXT_PUBLIC_SUPABASE_ANON_KEY` precisa ser publishable** (`sb_publishable_*` ou anon JWT) — usar secret no browser quebra com `Forbidden use of secret API key`.
- **Engine só envia quando `campaigns.status` ∈ {`approved`, `active`}.** Aprovação só via rota server-side `/api/campaigns/approve`.
- **Trava de 1 e-mail por campanha por execução do cron** — não remover sem substituir por rate limiter mais robusto.
- **`opted_out`** = compliance (descadastro/remoção). **`negative`** = recusa comercial. Os dois agregam como "Negativos" na UI mas têm semântica diferente.
- **Landing pública é intocável** salvo pedido explícito do usuário.

## Como rodar

```bash
npm install
npm run dev    # http://localhost:3000
npm run lint   # passa com warnings antigos da landing
npm run build  # passa
```

Variáveis sensíveis vivem em `.env.local`. Sem `SUPABASE_SERVICE_ROLE_KEY` o envio de teste e a aprovação de campanha respondem com erro orientando configurar.
